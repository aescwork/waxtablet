Introduction

Waxtablet is a terminal or command-line application for creating, reading, updating, deleting and displaying text information. The text information is organized into and stored as Entries which are stored in one or more databases. It was specifically written to serve as a journal program, but it can also be used for documentation, notes or general organization of information.

The text information -- the Entry -- is entered into a plain Hyper Text Markup Language (html) file, called "input.html". This file can be located anywhere on the computer. When the information is done being entered, input.html can then be read by waxtablet, the information in the file is parsed and stored in the current Entry database.

waxtablet treats an Entry as a distinct unit-grouping of information. An entry consists of:

  1. An Entry Number.
  2. Entry Text.
  3. Entry Keywords.
  4. An Entry Date.
  5. The name of the database (hereafter called the "entry-database") file where the Entry data is stored.

input.html is used not only for creating a new Entry, but also for modifying existing Entries. This will be discussed below.

Entries in the current entry-database can be selected according to certain criteria, such as Entry number, date range, text search and more.

Entries can also be updated. For this, they can be selected according to certain criteria as mentioned above. When they are selected, they are written to input.html, which can then be modified.

Entries can also be deleted, again according to certain search criteria.

Installation

Preliminaries

In order to use waxtablet, python version 2.6 or higher must be installed.

In order to install the application, the pip package manager must be installed.

Install

  1. Download the code from github here or use git in the terminal:

    git clone https://github.com/aescwork/waxtablet

  2. Navigate into the dist directory.
  3. Linux/Unix users, run:

    sudo pip install waxtablet-1.0.tar.gz (The waxtablet version number you see may be different.)

  4. Windows users, run:

    pip install waxtablet-1.0.tar.gz (The waxtablet version number you see may be different.)

  5. Pip will install waxtablet in the default location for applications which are available to all users.
  6. Navigate back up a directory and into the scripts directory.
  7. Use the install_additional_files.py script to copy over the directories and their files used by waxtablet in your home directory. Run:

    python install_additional_files.py

    Microsoft Windows users should see the waxtablet directory in their Users\<username> directory. Linux/Unix users should see the .waxtablet directory by navigating to their home directory and running

    ls -al

  8. Test the waxtablet installation. Run:

    waxtablet -M

    The Settings Main Menu should now display.

Getting Started

Once installation is complete, all of the directories and files needed for waxtablet are created and installed:

  • wtab_usr_data, containing:
    • wtab_app
  • html-templates, containing:
    • default.html
  • entry-databases
  • io_files, containing:
    • input.html
    • output.html
    • default.css
    • media_queries.css

Before using the application, create an entry-database where Entries will be stored. Use the following command:

waxtablet -N my-new-entry-database

Next, set this newly created entry-database to the current database:

waxtablet -C my-new-entry-database

waxtablet is now ready to create, read, update and delete Entries. (See below for how to do these.)

    Combining Options and Arguments in a Single Command

    Generally, different options can be combined together which execute different functions of the application. The following example sets a new entry-database to use, displays all the keywords in the current entry-database and create a new Entry in the current entry-database:

    waxtablet -N "personal" -l -v c

    However, Updating, Reading and Deleting Entries must be done with separate commands. The following example attempts to first retrieve Entries for display by searching for keywords, then Update Entries by group and then Delete some Entries by group:

    waxtablet -v r -e "python computers mobile" -v u -g "1 2 5 7" -v d -g "3 4 9 11"

    This won't work. The only thing the above command will do is retrieve the Entries for display and then exit. The Update and Delete functions won't be executed. This is because once a Read, Update or Delete Entries function is called, the action is executed and the application then immediately exits.

    The above commands would each need to be run separately:

    waxtablet -v r -e "python computers mobile"

    waxtablet -v u -g "1 2 5 7"

    waxtablet -v d -g "3 4 9 11"

    In any case, It is recommended that Reading, Updating and Deleting Entries by run as commands which execute those functions only and don't call any other options. While the application can otherwise handle calls to multiple options, calling too many options together in a single command might result in unexpected behavior.

    Creating Entries

    To create an entry, first make sure that the input.html file is set up properly so waxtablet can parse the Entry text. This is most easily done by first closing the input.html file, then entering the following command:

    waxtablet -T

    This outputs or writes the basic template to input.html which provides markup for structuring the Entry data.

    The template looks like this:

    <!-- BEGIN ENTRY -->
    <!-- NUMBER -->
    <!-- DATE -->
    <!-- KEYWORDS: -->
    <!-- TEXT -->
    <p></p>
    <p></p>
    <p></p>

    Once this is in input.html, open this file and add the Entry text.

    The first 5 lines contain only html comments (the lines with the exclamation mark). The only line which needs to be filled out by the user is the KEYWORDS: line. After the "KEYWORDS:", enter any keywords or topic-words for this Entry, like so:

    <!-- KEYWORDS: python text html computer-application command-line -->

    waxtablet will parse these and put them in the current entry-database for this entry. Keywords must be separated ONLY by a single space. Do not separate keywords with commas, dashes or any other character. If a keyword is to actually consist of two words, use a dash or any other text character between the two words.

    Leave the other comment lines there, and leave them alone.

    Below the comment lines are html tags. The tags shown above are for paragraphs (<p></p>).

    If you don't know html, just put all of the words you write between one or more sets of the paragraph tags like so:

    <p>This is a paragraph. This is a sentence of a paragraph. Here is another sentence in this paragraph.</p>

    <p>This is a sentence of another paragraph. We are now working on our second sentence of the second paragraph.</p>

    All of the text information for the Entry must be placed below the <!-- TEXT --> comment line. Any other text in valid html markup -- such as divs, sections, lists, tables, etc -- may of course also be added.

    When all the text has been entered, close the input.html and enter the following at the prompt:

    waxtablet -v c

    This will make waxtablet read the information from input.html into an Entry in the current database. Other relevant information indicated by the comment lines in above.

    Reading Entries

    Retrieving (or Reading) Entries involves reading the Entry information for one or more Entries from the current entry-database.

    The particular way in which Entries are targetted for selection is explained here. The search criteria which apply for retrieving Entries also apply to Deleting and Updating Entries.

    Each command described below for reading Entries involves using the -v option followed by a single letter argument r (for "read") followed by one of several other option letters which specify how to select Entries.

    Read Commands

    Select all of the Entries in the current entry-database:

    waxtablet -v r -a

    Target a range of Entries based on Entry Number. Put the two numbers in quotes, separated by a single space. The below demonstrates retrieving Entries whose numbers are 2 through 12.

    waxtablet -v r -n "2 12"

    Target a range of Entries based on Entry Date. Put the two dates in quotes, separated by a single space. The dates must be in the format YYYY-MM-DD. Example, search all Entries whose dates are from March 7, 2017 to June 12 2017:

    waxtablet -v r -n "2017-03-07 2017-06-12"

    Target the first number of Entries (by Entry Number). Optionally specify a number. Example, get the first 7:

    waxtablet -v r -j 7

    ...or just pass the option to get the default number (5):

    waxtablet -v r -j

    Target the last number of Entries (by Entry Number). Optionally specify a number. Example, get the first 7:

    waxtablet -v r -t 7

    ...or just pass the option to get the default number (5):

    waxtablet -v r -t

    Target Entries which have certain keywords. List all keywords together surrounded by quotes, each separated by a single space:

    waxtablet -e "python text html computer-application command-line"

    Target Entries which have a specific date. Dates must be formatted like YYYY-MM-DD:

    waxtablet -f "2017-03-07"

    Target Entries which have a certain value for a date-member, that is year, month or day:

    Get all Entries with a specific year:

    waxtablet -m "year 2017"

    Get all Entries with a specific month:

    waxtablet -m "month 03"

    Get all Entries with a specific day (date):

    waxtablet -m "day 17"

    Target Entries which have certain words in the text:

    waxtablet -s "python application text parse"

    Target Entries by one or more Entry numbers:

    (Get Entries with number 1, 4, 8, 11, 13, 21)

    waxtablet -g "1 4 8 11 13 21"

    The targetting of Entries also allows a strict query, which stipulates that all Entries must have each search value which are passed in as an argument to options which involve searching for more than one value.

    For example, the line above which demonstrates a search for certain keywords will return Entries which have one or more of any of the keywords passed in. Including the strict query option, however, makes waxtablet return only those Entries which have all of the passed-in keywords.

    This example below demonstrates. It passes the r argument to the -v option, which retrieves Entries from the database, the -e option followed by a list of words means to search for Entries with the keywords which follow it, and the -q option specifies that only Entries with all of the passed-in keywords should be retrieved:

    waxtablet -q -v r -e "python text html computer-application command-line"

    Note that, as shown above, the -q option must be passed-in the same command with the other options and arguments to specify the strict search.

    Deleting Entries

    Deleting Entries involves deleting the Entry information for one or more Entries from the current entry-database.

    The syntax described above for reading Entries also works for deleting them. The -v d option and argument are used instead.

    Below are two examples to demonstrate.

    Target all of the Entries in the current entry-database for deletion.

    waxtablet -v d -a

    Target a range of Entries based on Entry Number for deletion. Put the two numbers in quotes, separated by a single space. The below demonstrates retrieving Entries whose numbers are 2 through 12.

    waxtablet -v d -n "2 12"

    ... and so on.

    Note that the two examples above are the same as the first two examples demonstrating reading Entries. The only difference is the argument used to the -v option.

    Important note

    While it is possible to use the syntax for targetting Entries to delete multiple Entries, the safest way to delete Entries is one at a time. This can be done with the -g option, as with reading Entries above. For example, to delete one Entry whose Entry Number is 3:

    waxtablet -v d -g 3

    In the case of multiple deletions, it is more laborious but also more safe to delete one Entry at a time in this manner.

    The next safest way to delete multiple Entries is to use -g with multiple Entry numbers:

    waxtablet -v d -g "3 4 5 9"

    Remember: once you delete it, it is for all intents and purposes, gone.

    Updating Entries

    Updating Entries involves Reading the Entry information for one or more Entries from the current entry-database. The Entries are printed to input.html, where the details of them can be modified. When the desired modifications are completed, the input.html file should be closed, and waxtablet is called again at the prompt to parse the Entry information, after which it is updated in the current entry-database.

    The syntax described above for reading and Deleting Entries also works for selecting them for update. The -v d option and argument are used instead.

    Below are two examples to demonstrate.

    It is possible to update Entry Keywords, Entry Text, or both. waxtablet will update these depending on the particular setting used for this in the wtab_app database, where the applications configuration settings are stored. The default setting is to update both Entry Keywords and Entry Text. Setting it to either Entry Keywords or Entry Text will cause waxtablet to actually execute the update of the one or the other only, not both. To set this behavior, use the following commands:

    Set waxtablet to update only Entry Keywords:

    waxtablet -i "keywords"

    Set waxtablet to update only Entry Text:

    waxtablet -i "text"

    Set waxtablet to update both Entry Keywords and Entry Text:

    waxtablet -i "text_and_keywords"

    The default setting is to update Entry Text and Entry Keywords.

    The syntax for selecting Entries for update is otherwise as above;

    Target all of the Entries in the current entry-database for updating.

    waxtablet -v u -a

    Target a range of Entries based on Entry Number for deletion. Put the two numbers in quotes, separated by a single space. The below demonstrates retrieving Entries whose numbers are 2 through 12.

    waxtablet -v u -n "2 12"

    ... and so on.

    Next, open input.html and modify the Entry Keywords and Entry Text as desired. When done, close this file.

    After this, commit the changes made in input.html with this command:

    waxtablet -U

    The above will update all of the Entries modified in input.html.

    Merging Entries With An Update

    It is possible to merge two or more updates into one by modifying input.html. When this is done, waxtablet can optionally delete the old Entry or Entries that were merged into the remaining Entry.

    For example, imagine that after running the command:

    waxtablet -v u -g "1 2 3"

    three Entries have been written to input.html for updating.

    The input.html would look like this:

    <!-- BEGIN ENTRY -->
    <!-- NUMBER: 1 -->
    <!-- DATE: 2017-03-17 -->
    <!-- KEYWORDS: web html css browser markup -->
    <!-- TEXT -->
    <p>This is a paragraph for an Entry about web development. </p>
    <p>This is another paragraph: talk about html, css, markup, etc.</p>
    <p>Here is a paragraph.</p>

    <!-- BEGIN ENTRY -->
    <!-- NUMBER: 2 -->
    <!-- DATE: 2017-03-18 -->
    <!-- KEYWORDS: -- webserver apache php lamp-stack -->
    <!-- TEXT -->
    <p>This paragraph talks about php. Php is a language for generating dynamic web content.</p>
    <p>Here we discuss apache. Apache is a web server. Php can run on the apache web server.</p>
    <p>In this paragraph we can put it all together: Linux, Apache, MySQL and PHP. That's the "lamp" stack.</p>

    <!-- BEGIN ENTRY -->
    <!-- NUMBER: 3 -->
    <!-- DATE: 2017-03-18 -->
    <!-- KEYWORDS: python desktop tkinter logging -->
    <!-- TEXT -->
    <p>This paragraph talks about python. Use it to write desktop applications.</p>
    <p>This paragraph talks about tkinter. tkinter is a gui library for python.</p>
    <p>Finally, we talk about logging. Yes, Python has its own support to handle applications writing to their log files. </p>

    What we could do is merge two or more Entries together; In this example, let's merge Entries 1 and 2. More specifically, we will merge Entry 2 into Entry 1.

    To do this, modify input.html so it now looks like this:

    <!-- BEGIN ENTRY -->
    <!-- NUMBER: 1 -->
    <!-- DATE: 2017-03-17 -->
    <!-- KEYWORDS: -- web html css browser markup webserver apache php lamp -->
    <!-- TEXT -->
    <p>This is a paragraph for an Entry about web development. </p>
    <p>This is another paragraph: talk about html, css, markup, etc.</p>
    <p>Here is a paragraph.</p>
    <p>This paragraph talks about php. Php is a language for generating dynamic web content.</p>
    <p>Here we discuss apache. Apache is a web server. Php can run on the apache web server.</p>
    <p>In this paragraph we can put it all together: Linux, Apache, MySQL and PHP. That's the "lamp" stack.</p>

    <!-- BEGIN ENTRY -->
    <!-- NUMBER: 3 -->
    <!-- DATE: 2017-03-18 -->
    <!-- KEYWORDS: python desktop tkinter logging -->
    <!-- TEXT -->
    <p>This paragraph talks about python. Use it to write desktop applications.</p>
    <p>This paragraph talks about tkinter. tkinter is a gui library for python.</p>
    <p>Finally, we talk about logging. Yes, Python has its own support to handle applications writing to their log files. </p>

    In the above example, Entry 2 is gone, and Entry 1 has the keywords and paragraphs from Entry 2. (Note: Leave the Entry Date alone: if you modify it, the modification won't be updated in the entry-database.) These two Entries have now been merged together as Entry Number 1.

    Now close input.html if it is open in a text editor.

    In order to effect this change in the entry-database, waxtablet must be set to automatically merge updates in the database. Run the command:

    waxtablet -A

    Then run:

    waxtablet -U

    To update the entry-database. This will update Entry 1 to have the Entry Text and Entry Keywords from Entry 2, and it will delete Entry 2. Now the database will have only Entry 1, 3... etc.

    Displaying Entries

    Entries are displayed using html. Waxtablet uses two html files for this: output.html and an html file used as a template for the actual markup structure of the display. That is, the markup which formats the presentation or appearance is all in the template. The template markup along with the information in the Entries is written to output.html. When this process is complete, output.html is then opened (typically in a browser) for display.

    So once all of the markup and Entries have been written to output.html, just open it in your browser by entering the full path to output.html in the browser bar or navigating the browser to the file's location in the html-templates directory.

    The templates are stored in the html-templates directory. Waxtablet initially includes a default template (along with two .css files), but any other templates can also be added to the html-templates directory and selected for use in display.

    Styling the html in the template

    Naturally, .css (and javascript) files can also be used with each template. Just remember to include them with <link> tags in the template's html.

    The Entries themselves are written to the output.html file surrounded by markup.

    <div class=\"entry\">

    <p class="entry_number"></p>

    <p class="entry_date"></p>

    <p class="entry_keywords"></p>

    </div>

    Working with html templates

    In order to select an html template to format the display, bring up the Settings Main Menu:

    waxtablet -M

    and select the option to "Set the current html template to be used for displaying entries". All templates in the html-templates are displayed.

    (NOTE: Do not place any files in html-templates other than actual templates. Waxtablet displays templates for the above selection based on what files it finds in the html-templates directory that have a .html extension in their filenames, and does not discriminate among them.)

    To add a template to html-templates, simply place the following comment-line wherever in the markup that the Entry information is to be added:

    <!-- INSERT ENTRIES -->

    (You can see an example of this by looking at the default.html template in html-templates.) Then copy or move the file manually to the html-templates directory.

    The location of the html-templates is placed in a default location when waxtablet is installed, but it can be moved elsewhere. This can be done by first moving the html-templates where you want it to go. Using the Main Settings Menu, select option 1, and enter the full or absolute path to the new location of the directory.

    On Microsoft Windows, an absolute path to the directory's location looks like:

    C:\Users\username\Documents\html-templates

    On Linux or Unix/Macintosh, an absolute path to the directory's location looks like:

    /home/username/Documents/html-templates

    Additional Options (Optional Arguments)

    -q ( --strict-query) "Target only entries whose specified columns have all corresponding search values. Use this option in the same command which selects Entries for display.
    -w ( --delete-keywords) Delete the keywords which match the words passed in as an argument following this option from the keywords table and all entries.
    -l ( --display-keywords) Display all keywords for the current entry database in the terminal.
    -A ( --auto-delete) If two or more entries are written to the input.html file for an update and two or more are then merged, delete the entry or entries which were merged into the other entry.
    -N ( --new-entry-db) Create a new Entry database. Ex. -N my-new-entry-database.
    -C ( --change-to-entry-db) Change to a another Entry database. Ex. -C my-other-database.
    -D ( --delete-entry-db) Delete an Entry database. Ex. -D my-new-entry-database.
    -S ( --show-all-entry-dbs) Show All entry-databases.
    -P ( --change-template) Change to a new html template for displaying the entry contents. (The template will written to output.html.)
    -L ( --show-all-templates) Show all html templates.
    -R ( --delete-template) Delete an html template.
    -M ( --settings-main-menu) Show a menu to set/initialize the values for settings in the application database.
    -T ( --input-template) Outputs (writes) basic information about how to structure and use the input.html file for creating entries to the input.html file.
